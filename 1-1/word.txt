IT基本用語
■ IT

IT　　（Information Technology/情報技術）
　　　　情報をデジタル（数値によって表現）で扱う・制御する技術


■ OS
OS　(Operating System)最も基本的な機能を制御しているソフトフェア（コンピュータを動かすプログラム）
　　　パソコンを使っていて当たり前だと思う操作を提供　クリック、マウスの動作
Windows　Microsoft社が開発
macOS 　Apple社が開発
(Mac OS X)
UNIX　　最も古いOS現在でも進化中で企業内や商品（カーナビや携帯電話）で使われる
Linux　　無料でカスタマイズが可能なプログラムのOS  ハードウェアに依存しない
　　　　　(世界中の有志で開発)


■ 言語・環境
マークアップ／フロントエンド　テキストにタグをつけてコンピュータが読み込みできるようにする、目に見える方
HTML  私たちが目で見ているwebページを記述するためのマークアップ言語
CSS　　要素の色や配置など装飾をするための言語
MySQL　オープンソースのデータベース管理システム 高機能で信頼高くて人気　有料と無料がある　（WordPressで採用）
PHP　　動的にwebサービスを作れるスクリプト言語
JavaScript　同的なWebページを作成できるプログラミング言語。HTMLコードの変更や、CSSスタイルの変更もできる。
jQuery Javascriptを使いやすいように拡張してくれているもの　どのブラウザでも同じコードで使える
Apache 世界一普及してるwebサーバソフトウェア（HTTP）サーバ
Bootstrap twitter社が開発したCSSのフレームワーク（よく使う機能をまとめた枠組み）


■ エンジニアなど
SE　　システムエンジニア　PMが制作した工程表に従ってこなす。担当のお客さんと打ち合わせて上流工程
PG（PE）プログラマー　SEの手足　SEが用意した仕様書をもとにプログラミング・単位テストをするのが主な作業
Webデザイナー　企業や依頼者の要求に応じたwebサイトのデザインを考案する


■ UI（ユーザーインターフェース）
UI　ユーザーとサービスの接点全てのこと。webサイトではユーザーが目にする、操作するもの全てのこと
CUI　Character-based 情報を提示したり受け付けたりするやり取りを文字によって行う
GUI　Graphic-based 情報の提示に画像や図形を使って、魔うるやタッチスクリーンなどで行う


■ ファイル／ディレクトリ
ファイル　写真動画、テキストなどまとまりを持った一つ一つのデータのこと
ディレクトリ　記録メディア上のファイルの保存場所　CUI フォルダと役割は似ている
ルート・ディレクトリ　最上部のディレクトリ（大元）
カレント・ディレクトリ　今いるディレクトリ
パス　　目的のファイルがどこのフォルダにあるかを示す
絶対パス　ルートディレクトリからの目的地の経路　urlでページ指定
相対パス　作業しているフォルダから目的地までの経路


■ 各種プロトコル
HTTP  webのデータ転送に用いられる用いられるプロトコル（通信規約）
FTP　　インターネットでファイル転送を行うことができるプロトコル
HTTPS　httpが暗号化されているもの。
SSL　　インターネット上のデータの通信を暗号化し、盗聴や改竄を防ぐプロトコル　httpsからはじまる
SSH  ネットワークに接続された機器を遠隔操作して管理するためのプロトコル
ポート、ポート番号　　IPアドレスを住所とするなら、何号室か？ということ アプリケーションを特定する
HTTP、FTP、SSHの各ポート番号　HTTP：80番 FTP：TCP20番とTCP21番　SSH:22番
ファイアウォール  企業など内部ネットワークを不正なアクセスから守るための“防火壁”
TCP　安全性重視の通信規約　IP（インターネット上で情報をやり取りする時の通信規約）の一段階上　HTTP、FTPの橋渡し


■ IPアドレス
IPアドレス　コンピュータ一台ごとに振り当てられた識別番号
グローバルIPアドレス　　インターネットに直に接続された機器に割り当てられるipアドレス
プライベートIPアドレス　　企業など組織の内部で運用されるネットワーク上で各機器に割り当てられるipアドレス


■ ドメイン
ドメイン　　インターネット上の住所。wabサイトだとhttps://の後に続く部分
DNS　　IPアドレスを人間がわかりやすいように変換してくれる仕組み
VPN　インターネット上に仮想的な専用線を設けて安全な経路でデータをやり取りする仕組み
ブリッジ　　受信したデータの宛先情報を解析して中継したり、破棄する。
プロキシサーバー　企業などの内部ネットワークとインターネットの境界にあり、内部のネットワークの代理としてインターネット上のコンミュータに接続するサーバソフトウェア


■ 電子メール
SMTP　メールを送信するための一般的な通信手段
POP　　メールを受信するための一般的な通信規約　メールを端末にダウンロード
IMAP　popと一緒で、違いはキャッシュのみ端末で管理　サーバに残す


■ セッションとクッキー
セッション　ネットワークに接続してから切断するまでの一連の操作や通信のこと
クッキー　　利用者の識別や属性に関するデータやアクセスした日時を利用者側に保存させる仕組み
キャッシュ　直近に読み込んだものや使用頻度が高いものを高速な記憶装置に複製すること


■ DB（データベース）について
インデックス　データをすぐに見つけられるような整理された所在情報
スロークエリ　処理速度が遅いSQL（データベースを操作するための言語）のこと　（クエリ：データの問い合わせや要求）
チューニング　　目標の状態に近づけたり性能を最大限に引き出すこと


■ テストについて
フロー　コンピュータプログラムを構成する命令や手続き、処理の流れのこと
単体テスト　プログラムの作成が終わった段階で、問題がないか不具合を確認する
結合テスト 複数のプログラムを同時に稼働させて問題がないか不具合を確認する


■ チケット管理ツール　　
チケット管理ツール　プロジェクトに必要な情報をまとめておくことができる管理システム
タスク　コンピュータ内で処理する仕事の単位　やること


■ バージョン管理
Git　　分散型バージョン管理システム　編集履歴を管理できる
SourceTree　バージョン管理システム　Gitとは違い、コマンドではなくてGUIによって直感的に操作が可能
SVN　　同じくバージョン管理システム　リモートリポジトリだけ
コンフリクト　変更履歴の競合　２人が同じファイルで作業をして１つのファイルに対して別々の変更点があると発生
デグレ　これまで正常に動いていた機能が動かなくなるトラブル
マージツール　ブランチで作成した複数の開発ラインを一つにまとめる方法
